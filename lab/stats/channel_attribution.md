# Channel attribution: is the BM25-only region advantage real or composition?

Final-wave conditioning analysis (2026-07). Generated by
`lab/stats/channel_attribution.py` → `lab/stats/channel_attribution.json`.

## Question

The BM25-only ablation arms showed a raw REGION-level advantage over the
shipped defaults on each arm's own file-correct subset (Lite FUNCTION 60.45
vs 57.76, LINE 49.25 vs 46.21; Verified FUNCTION 51.80 vs 50.53, LINE 41.83
vs 38.50). But the defaults recover more files (Lite 277 vs 268, Verified 374
vs 361), so the subsets differ: if the channel-recovered extra files are
harder to localize within, the defaults' region averages are dragged down by
composition, not by channel-caused region damage.

## Method

Restrict BOTH arms of each pair to the INTERSECTION of instances that are
file-correct in both arms, then recompute FUNCTION-exact and LINE
(all-or-nothing) correct-rates on that identical instance set. Per-instance
signals use the scorers' own conventions (FILE = `all_gold_files_retrieved`,
LINE = `hunk_line_recall == 1.0`, FUNCTION = scorer detail flag).

## Conditioned tables

### SWE-bench Lite 300 (defaults `full300_v11` vs `lite300_bm25`), intersection n=268

| metric | defaults | bm25-only | delta (defaults − bm25) | discordants (defaults-only : bm25-only) |
|---|---|---|---|---|
| FUNCTION | 58.96 (158/268) | 60.45 (162/268) | −1.49pp | 4 : 8 |
| LINE | 47.01 (126/268) | 49.25 (132/268) | −2.24pp | 2 : 8 |

BM25's file-correct set is an exact subset of the defaults' here (268 of 268

## Verdict

**Mostly, but not entirely, a composition effect — and the residual is
LINE-only.**

- FUNCTION: conditioned on identical instances the gap collapses to noise
  (−1.49pp with 4:8 discordants on Lite; −0.28pp with 8:9 on Verified). The
  raw FUNCTION advantage of the BM25 arm was composition: the channels'
  extra recovered files (9 on Lite, ~14 on Verified) are harder to localize
  within, and they sit only in the defaults' denominator.
- LINE: a small consistent residual survives conditioning (−2.2pp on both
  benches, discordants 2:8 and 2:10 — same sign, same size). On the same
  file-correct instances, running the extra channels slightly dilutes exact
  line coverage — plausibly extra channel candidates displace budget from
  the gold file's regions during packing. This residual is below the paired
  significance bar on the full sets (`lab/stats/*_defaults_vs_bm25.json`:
  LINE McNemar p=0.39 Lite / p=0.12 Verified), so it is a lead, not a
  finding.
- The channels' unambiguous contribution is FILE recall: −3.00pp (McNemar
  p=3.9e-03) on Lite and −3.19pp (p=9.8e-04) on Verified when they are
  turned off — the only significant deltas in the wave's paired tests.

Net: keep the channels; their FILE gains are real and significant, their
region cost on already-found files is ~0 at FUNCTION level and ≤2pp
(non-significant) at LINE level. The raw BM25 region advantage should not be
quoted without this conditioning.
